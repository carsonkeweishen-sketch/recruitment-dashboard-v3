# 02｜Recruitment Dashboard v3 Phase 路线图

## 总原则

v2 正式按 Phase 0 - Phase 13 推进。每个 Phase 只做一个明确范围，完成后必须自检、build、commit、审查。

---

## Phase 0：工程底座与持久化机制

范围：Next.js App Router、TypeScript、Tailwind CSS、Prisma、PostgreSQL 配置、Git 初始化、README、`.env.example`、docs 初始目录、AppShell、Sidebar、Topbar、Design Token、基础 UI 组件、`/api/health`。

验收：typecheck / lint / build 通过，有 commit hash，不做业务页面。

## Phase 1：权限、角色与系统边界

范围：admin / leader / hrbp / recruiter / business_owner / interviewer，权限矩阵，scope，mock role switcher，服务端权限框架，Moka 边界文档。

验收：不是前端假权限，API 权限可复用，每个角色范围清晰。

## Phase 2：核心数据模型与 Seed

范围：User、Department、Job、Candidate、Application、Interview、InterviewFeedback、ActionItem、OfferRisk、OfferCommitment、ProfileCalibration、BusinessFeedback、AiAnalysisLog、ReportSnapshot、ImportBatch、ActivityLog。

验收：Prisma migrate 正常，seed 使用真实感虚拟数据，无真实隐私，不 destructive migration。

## Phase 3：岗位管理与 JD / 画像体系

范围：`/jobs`，岗位列表、岗位详情、JD、职级、部门、Owner、岗位漏斗、岗位画像、基础卡点信号。

## Phase 4：候选人与投递链路

范围：`/candidates`、`/applications`，候选人档案、投递记录、阶段、来源、多岗位复用、人才池沉淀。

## Phase 5：业务筛选反馈与岗位画像校准

范围：BusinessFeedback、拒绝原因统计、画像校准、ProfileCalibration、Timeline、画像偏差信号。

## Phase 6：面试反馈与结构化面评

范围：`/interviews`，InterviewFeedback，6 大评分维度，推荐结论，证据记录，多轮面试汇总，面试质量信号。

6 大维度：role_competency、business_understanding、problem_solving、communication、ownership_collaboration、motivation_stability。

## Phase 7：协同 Action 任务中心

范围：`/actions`，ActionItem，来源信号，责任人，状态流转，逾期，ActivityLog。

信号：LOW_EFFECTIVE_CANDIDATES、STALLED_HR_SCREEN、STALLED_BUSINESS_SCREEN、FEEDBACK_OVERDUE、EVIDENCE_INSUFFICIENT、SCORE_INCONSISTENT、CALIBRATION_NEEDED、CANDIDATE_REDIRECT。

## Phase 8：数据导入与历史数据接入

范围：`/imports`，Excel / CSV，字段映射，数据校验，确认导入，导入历史。

## Phase 9：Offer 风险与入职前承接

范围：`/offer-risks`，OfferRisk、OfferCommitment、风险类型、承诺台账、Deadline、ActivityLog、Action 联动。

风险类型：SALARY_GAP、COMPETING_OFFER、DECISION_DELAY、START_DATE_UNCERTAIN、COMMITMENT_UNRESOLVED、PRE_ONBOARDING_SILENCE、EXPECTATION_MISMATCH。

## Phase 10：AI 辅助分析系统

范围：AI Provider、OpenAI / DeepSeek Adapter、脱敏、AiAnalysisLog、promptVersion、inputHash、fallback、cache、timeout、rate limit、权限过滤。

能力：resume_analysis、candidate_recommendation、interview_performance、interviewer_quality、job_bottleneck、ai_qa。

## Phase 11：招聘复盘与周报

范围：`/reports`，ReportSnapshot，周报预览，生成快照，历史列表，详情，确认归档，ActivityLog，AI Usage Summary。

闭环：Preview -> Generate Snapshot -> List -> Detail -> Confirm -> ActivityLog。

## Phase 12：Design System、UI Polish 与开源化收口

范围：Design Token、AppShell、UI Components、UI Gate、README、docs、`.env.example`、安全检查、API 文档、权限文档、Moka 边界文档、Docker 可选。

## Phase 13：面试官赋能中心

范围：`/interviewer-enablement`，面试题库、岗位题库、职级题库、面试轮次题库、追问题库、笔试题生成、评分 Rubric、面试官质量评估、成长建议。
