## 使用顺序

不要一次性把所有历史上下文塞给 WorkBuddy。第一轮只给：

- 00_先读我_AI协同与执行规则.md
- 02_Phase路线图_0到13.md
- 08_Phase0_WorkBuddy执行任务书.md
- 09_Phase自检报告模板.md
- 10_给WorkBuddy的首次消息.md

其余文档让 WorkBuddy 写入 repo 的 docs 目录，并在需要时引用。

## 核心原则

- v2 不缩水；
- 正式按 Phase 0 - Phase 13 推进；
- 每个 Phase 必须 commit；
- 每个 Phase 必须 build；
- 每个 Phase 必须自检；
- 外部审查通过后才进入下一 Phase。
