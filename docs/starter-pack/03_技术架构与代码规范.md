# 03｜技术架构与代码规范

## 1. 固定技术栈

- Next.js App Router
- TypeScript
- PostgreSQL
- Prisma
- Tailwind CSS
- 自建 UI primitives
- OpenAI / DeepSeek Provider Adapter
- xlsx / csv-parse
- Recharts / ECharts 可后置

不要随意更换技术栈。

## 3. 分层原则

App 层负责页面路由、API route、组装 UI。  
Components 层分为 layout、ui、domain。UI 组件禁止写死招聘业务字段。  
Server 层负责权限、聚合、业务服务、AI、导入校验、报表快照、Action 规则。  
Prisma 层只负责数据结构和关系，不塞复杂逻辑。

---

## 4. API 规范

所有 API 返回：

```ts
type ApiResponse<T> = {
  success: boolean;
  data?: T;
  error?: {
    code: string;
    message: string;
  };
};
```

错误码建议：UNAUTHORIZED、FORBIDDEN、VALIDATION_ERROR、NOT_FOUND、CONFLICT、AI_NOT_CONFIGURED、AI_PROVIDER_ERROR、INTERNAL_ERROR。

---

## 5. 权限调用规范

所有服务端 API 必须经过：

```ts
validateAndBuildContext()
requirePermission()
buildScopeWhere()
```

禁止：只靠前端隐藏；API 返回全量再前端过滤；business_owner 看全量；interviewer 访问 reports / ai / imports。

---

## 6. 数据库迁移规范

禁止：

```bash
prisma db push --accept-data-loss
```

必须使用：

```bash
pnpm prisma migrate dev --name phase_x
```

每次 schema 改动必须 migrate、seed、build、commit、自检报告说明。

---

## 7. package scripts

必须包含：

```json
{
  "dev": "next dev",
  "build": "next build",
  "lint": "next lint",
  "typecheck": "tsc --noEmit",
  "db:migrate": "prisma migrate dev",
  "db:seed": "prisma db seed",
  "db:reset": "prisma migrate reset"
}
```

---

## 8. Git 规范

每个 Phase 必须：

```bash
git status
git add .
git commit -m "phase-x: ..."
git tag phase-x-complete
```

自检报告必须包含 commit hash。
