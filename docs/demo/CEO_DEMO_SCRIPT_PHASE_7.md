# CEO Demo Script — Phase 7: 招聘风险行动中心

> 时长：3-5 分钟
> 目标：展示系统如何主动发现和推动解决招聘流程中的风险与卡点

---

## 演示路径

### Step 1: 首页全景（30 秒）

> "这是招聘风险行动中心。我们不再需要 HR 手动翻邮件、查 Excel、催微信来找问题。"

展示：
- 页面标题"招聘风险行动中心"
- KPI 卡片：7 个待处理、3 个逾期、4 个高优
- 说话点：系统自动从面试反馈、候选人状态、岗位流程中识别风险，生成行动项

### Step 2: 逾期筛选（20 秒）

> "点一下「只看逾期」，立刻看到所有超时的风险事项。"

展示：
- 逾期筛选激活
- 说话点：每个逾期项都有截止时间和负责人，不需要追问"谁在跟"

### Step 3: 行动详情 — 概览（30 秒）

> "点开一条，看详情。"

展示：
- Drawer Overview Tab
- 说话点：一眼看到这个行动是什么、为什么产生、谁负责、什么时候到期

### Step 4: 关联信息（20 秒）

> "切到关联信息，能看到这个行动关联了哪个岗位、哪个候选人、面试到哪一步。"

展示：
- Linked Context Tab
- 说话点：不是孤立的待办，是有上下文的招聘决策辅助

### Step 5: 活动记录（20 秒）

> "活动记录记录了每一步操作，谁在什么时候做了什么。"

展示：
- Activity Tab
- 说话点：可复盘、可审计

### Step 6: 闭环操作（30 秒）

> "处理完成后，可以标记已解决，写处理说明。或者忽略，写忽略原因。所有操作都记录在活动时间线里。"

展示：
- Resolve Modal（含操作说明）
- 或 Dismiss Modal（含操作说明）
- 说话点：关闭行动不会自动改变候选人状态，处理说明会记录下来

### Step 7: 权限控制（15 秒）

> "不同角色看到的内容不同。面试官只能看到跟自己相关的。"

展示：
- 切换 interviewer 角色
- 说话点：信息权限受控，不会泄露
