# CEO Demo Q&A — Phase 7: 常见问题预答（Final Lock 版）

## Q1：这个和 Moka 有什么区别？

Moka 偏招聘流程和 Offer 管理；这个系统偏招聘判断、风险识别和协同闭环。它不替代 Moka，而是补 Moka 之上的招聘经营分析层。

## Q2：这是不是替代 HR？

不是。它把 HR 靠记忆追的风险和卡点系统化，让 HR 从催流程转向判断、沟通和业务协同。

## Q3：AI 在哪里？

当前先把招聘过程结构化，AI 后续会做受控辅助，例如风险识别、追问建议、复盘总结，但不会自动录用或淘汰候选人。

## Q4：数据从哪里来？

来自招聘过程本身：岗位、候选人、投递、面试反馈、业务反馈和行动记录。后续可以接 Moka 或飞书，但本阶段先打通核心闭环。

## Q5：面试官会不会觉得被监控？

不会。系统不做面试官排名，也不做羞辱榜，只帮助团队看反馈是否及时、证据是否充分、风险是否闭环。

## Q6：怎么保证数据安全？

按角色权限控制，敏感字段不展示，关键操作留痕，错误态和权限态不泄露对象信息。

## Q7：现在能上线到什么程度？

可以作为内部小范围试用版本，先用于招聘风险行动闭环和面试反馈质量改进，后续再逐步接入更多系统。

## Q8：下一步最有价值的是什么？

把 Action 嵌入岗位、候选人和 Dashboard，形成从岗位到候选人再到风险行动的完整招聘经营视图。
