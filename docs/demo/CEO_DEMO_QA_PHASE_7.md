# CEO Demo QA — Phase 7: 常见问题预答

## 数据问题

**Q: 这些数据是真实的吗？**
A: 演示数据是基于理然业务结构构造的脱敏演示数据。所有岗位、部门、候选人名称均为虚拟构造，不包含真实个人信息。系统接真实 PostgreSQL 数据库，API 全部真实调用。

**Q: 能接我们现有的招聘系统吗？**
A: 当前阶段已预留 Moka/飞书集成接口（代码已有 client），但未激活。后续可按需接入。

## 功能问题

**Q: 行动项是谁创建的？**
A: 两种方式：系统规则自动生成（基于反馈逾期、风险信号等）和手动创建。演示中两种都有。

**Q: 解决和忽略有什么区别？**
A: 解决 = 问题已处理，记录处理说明。忽略 = 问题不再适用（如重复、线下已处理），记录忽略原因。两者都不会删除记录。

**Q: 会不会自动改变候选人状态？**
A: 不会。行动项是独立的跟进机制，不影响候选人阶段或岗位状态。

## 权限问题

**Q: 谁可以看到所有行动？**
A: 管理员和 Leader 可以看到全部。HRBP 看本部门。招聘专员看自己负责的。面试官只看与自己相关的。

**Q: 数据安全吗？**
A: 不存储手机号、邮箱、身份证等敏感信息。API 有 Scope Guardrail 权限控制。

## 后续规划

**Q: 下一步做什么？**
A: Phase 7.5 将支持 Job Detail 和 Candidate Detail 页面中嵌入关联 Actions Tab，让招聘人员在查看岗位/候选人时直接看到相关风险。
