# 理然招聘模型

> 基于理然招聘项目分析模型 v3.2 精简实战版 + 岗位 SOP 提取。
> 用于 Phase 3 岗位管理，Phase 5 画像校准，Phase 11 周报复盘。

## 招聘分析模型 v3.2 核心框架

### 岗位拆解规则

1. **岗位画像建模**：mustHave（硬性）+ niceToHave（加分）+ 目标公司 + 行业偏好
2. **渠道策略**：猎头/内推/平台按岗位级别分渠道
3. **招聘漏斗**：sourced → hr_screen → business_screen → first_interview → second_interview → final_interview → offer_risk → hired

### 岗位画像校准规则

- 画像校准触发：连续 3 个候选人因同一原因淘汰
- 校准内容：mustHave 调整、薪资区间修正、目标公司扩展
- 记录 beforeSnapshot → afterSnapshot
- 业务负责人主导，HRBP 复核

### 候选人评估口径

- 6 维度评分（Phase 6 展开）
- 结构化面试反馈（Phase 6 展开）

### 面试反馈口径

- 面试官评分一致性检查
- 证据记录要求
- Phase 6 展开

### Offer 风险口径

- 7 大风险类型
- Phase 9 展开

### 招聘复盘口径

- 周报 KPI 体系
- Phase 11 展开

## 招聘流程 SOP 摘要

从岗位 SOP（BP/招聘负责人/招聘/SSC）提取：

| 阶段 | 关键动作 | 责任人 |
|------|----------|--------|
| 需求确认 | 新增/补充/汰换岗位画像确认，优先级排序 | HRBP + 部门负责人 |
| 计划制定 | 年度编制计划，月度滚动更新 | 招聘负责人 |
| 渠道管理 | 猎头/内推/平台分发 | 招聘 |
| 过程管理 | 筛选→面试→Offer→入职全流程 | 招聘 |
| 入职承接 | 合同/花名册/IT开通/入职指引 | SSC |
| 复盘归档 | 招聘数据汇总，画像校准 | HRBP |

## Phase 3 使用范围

- 岗位列表页展示理然真实部门+岗位+职级
- 岗位详情页展示 JD（职责+要求+加分）
- 岗位画像字段（mustHave/niceToHave）
- 基础漏斗信号（Phase 5 后展开）
