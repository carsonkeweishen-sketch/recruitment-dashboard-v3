# Drawer / Modal 交互规范

> 版本：v1.0
> 生效日期：2026-06-27
> 适用范围：所有 Drawer 和 Modal 组件
> 关联文档：`docs/design/RIJON_RECRUITMENT_DESIGN_SYSTEM.md`
> 参考：Linear Side Panel / Stripe Modal / Greenhouse Scorecard

---

## 1. 概述

Drawer 和 Modal 是本系统最重要的信息分层工具。核心原则：

- **Drawer**：查看详情、多信息区块
- **Modal**：表单提交、确认操作
- **Tabs**：Drawer 内信息分组

---

## 2. Drawer 规范

### 2.1 宽度三档

| 档位 | 宽度 | 使用场景 | 示例 |
|------|------|---------|------|
| **窄** | `w-[480px]` | 简单详情、单信息源 | 面试反馈查看 |
| **标准** | `w-[640px]` | 多信息区块、标准详情 | 候选人详情、岗位详情 |
| **宽** | `w-[720px]` | 含 AI 分析、复杂关联 | 候选人 AI 分析、岗位全貌 |

### 2.2 结构

```
┌──────────────────────────────┐
│ 固定头部（sticky top-0 z-10）   │
│ 标题 + 状态标签 + 关闭按钮 [✕]  │
├──────────────────────────────┤
│                              │
│ Tabs 导航                     │
│ [概览] [画像] [漏斗] [反馈] [动态]│
│                              │
├──────────────────────────────┤
│                              │
│ Tab 内容区（可滚动）            │
│                              │
│                              │
├──────────────────────────────┤
│ 底部操作栏（可选）              │
│ [取消]              [确认]    │
└──────────────────────────────┘
```

### 2.3 交互规则

| 规则 | 说明 |
|------|------|
| **进入方向** | 从右侧滑入（`animate-in slide-in-from-right`） |
| **遮罩** | 黑色半透明 `bg-black/30` |
| **ESC 关闭** | 支持键盘 ESC 关闭 |
| **点击遮罩关闭** | 点击遮罩关闭（除非有未保存内容） |
| **有未保存内容** | 关闭前弹出确认提示："有未保存的更改，确定关闭？" |
| **body scroll** | Drawer 打开时禁用 body 滚动 |

### 2.4 Tabs 规则

| 规则 | 说明 |
|------|------|
| **数量上限** | 最多 5 个 Tab |
| **标签类型** | 名词（概览/画像/反馈），不用动词（查看/编辑） |
| **加载方式** | 数据预加载，Tab 切换不需要 loading |
| **默认 Tab** | 打开时默认显示第一个 Tab |
| **信息粒度** | 不同粒度的信息分不同 Tab |

### 2.5 禁止

- ❌ 不堆 raw JSON
- ❌ 不展示敏感信息（手机号脱敏为 `138****1234`，邮箱脱敏为 `li***@example.com`，薪资不展示具体数字）
- ❌ Drawer 内不套 Modal
- ❌ 不显示技术错误（Prisma error, SQL, stack trace）

## 3. Modal 规范

### 3.1 宽度三档

| 档位 | 宽度 | 使用场景 | 示例 |
|------|------|---------|------|
| **小** | `max-w-sm` (384px) | 简单确认 | 确认删除、确认忽略 |
| **中** | `max-w-md` (448px) | 简单表单 | 新建候选人 |
| **大** | `max-w-lg` (512px) | 复杂表单 | 面试反馈、画像校准 |

### 3.2 结构

```
┌──────────────────────────────┐
│ 固定头部                      │
│ 标题 + 关闭按钮 [✕]           │
├──────────────────────────────┤
│                              │
│ 表单内容区                    │
│ ┌────────────────────────┐  │
│ │ 字段1                  │  │
│ │ 字段2                  │  │
│ │ ...                    │  │
│ └────────────────────────┘  │
│                              │
├──────────────────────────────┤
│ 底部操作栏                    │
│ [取消]            [提交/确认] │
└──────────────────────────────┘
```

### 3.3 交互规则

| 规则 | 说明 |
|------|------|
| **位置** | 居中弹出，顶部偏移 `pt-[10vh]` |
| **遮罩** | 黑色半透明 `bg-black/30` |
| **ESC 关闭** | 支持键盘 ESC 关闭 |
| **点击遮罩关闭** | 不关闭（防止误操作丢失输入） |
| **有未保存内容** | 关闭前弹出确认提示 |
| **提交按钮** | 必须有 loading 状态（"提交中..."） |
| **成功** | toast 通知，自动关闭 Modal |
| **失败** | 错误信息在 Modal 内展示，Modal 不关闭 |
| **表单校验** | 必填项即时校验，不等到提交时才校验 |

### 3.4 一个 Modal 一个动作

| 允许 | 禁止 |
|------|------|
| 新建候选人 | 新建候选人 + 同时分配岗位 |
| 提交面试反馈 | 提交反馈 + 同时推进阶段 |
| 确认删除 | 确认删除 + 同时发送通知 |

多步骤操作使用 **StepIndicator**，不是嵌套 Modal。

### 3.5 错误处理

| 错误类型 | 展示位置 | 文案示例 |
|---------|---------|---------|
| 表单校验错误 | 字段下方 | "请输入候选人姓名" |
| API 业务错误 | 表单顶部 | "该候选人已存在" |
| 网络错误 | 表单顶部 | "网络异常，请稍后重试" |
| 权限错误 | Modal 关闭 + 页面展示 | — |

**禁止暴露**：
- ❌ Prisma error
- ❌ SQL 语句
- ❌ stack trace
- ❌ 数据库连接信息
- ❌ API Key
- ❌ 内部 ID

## 4. Tabs 规范

### 4.1 使用场景

| 页面 | Tab 数量 | Tab 列表 |
|------|---------|---------|
| 岗位详情 Drawer | 5 | 概览 / 画像 / 漏斗 / 反馈 / 动态 |
| 候选人详情 Drawer | 4 | 概览 / 投递 / 面试 / 动态 |
| 面试详情 Drawer | 4 | 概览 / 反馈 / 质量 / 风险 |

### 4.2 规则

| 规则 | 说明 |
|------|------|
| **标签名词** | 概览、画像、漏斗 — 不用"查看概览"、"编辑画像" |
| **默认 Tab** | 第一个 Tab |
| **切换无 loading** | 数据预加载 |
| **Tab 数量** | 不超过 5 个，超出时合并或拆分子页面 |
| **信息粒度** | 不同粒度分开 |

---

## 5. 信息分布决策树

```
需要展示信息
├── 1-3 个字段 → 列表行内展示
├── 4-8 个字段 → 列表行 + Tooltip
├── 8-15 个字段 → Drawer（单 Tab）
├── 15+ 个字段 → Drawer（多 Tabs）
├── 需要用户输入 → Modal
├── 需要用户确认 → Modal（sm）
└── 需要分步操作 → Modal + StepIndicator
```

---

## 6. 禁止事项总览

| # | 禁止 | 原因 |
|---|------|------|
| 1 | Drawer 内套 Modal | 层级混乱 |
| 2 | Modal 里塞多个步骤 | 用 StepIndicator |
| 3 | 暴露技术栈 | Prisma error / SQL / stack trace |
| 4 | 暴露敏感字段 | 手机号 / 邮箱 / 薪资 / API Key |
| 5 | raw JSON dump | 无业务语义 |
| 6 | Tab 用动词 | 用名词 |
| 7 | Tab 超过 5 个 | 合并或拆分 |
| 8 | Modal 不设 loading | 防止重复提交 |
| 9 | 表单错误用 toast | 应在 Modal 内展示 |
| 10 | 关闭 Modal 丢输入 | 有未保存内容需确认 |
