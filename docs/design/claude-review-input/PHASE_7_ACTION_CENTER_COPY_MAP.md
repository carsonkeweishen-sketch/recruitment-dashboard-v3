## 五、Drawer 内文案

### Tab 名称
- 概览
- 关联信息
- 活动记录

### Overview Tab
- 基础信息（标题 + 描述）
- 处理状态（分类 / 状态 / 优先级 / 来源）
- 责任人与来源（负责人 / 创建人 / 来源说明）
- 时间信息（截止时间 / 创建时间）
- 处理说明 / 忽略原因（已解决/已忽略时显示）

### Linked Context Tab
- 关联岗位（岗位名称 + 编号）
- 关联候选人（姓名）
- 关联投递（阶段 + 候选人）
- 关联面试（轮次 + 面试官）

### Activity Tab
- 事件类型标签：创建 / 已解决 / 已忽略
- 时间 + 操作人
- 详情（JSON detail 字段原文）
- 空状态：暂无操作记录

### Drawer Footer
- 标记为已解决（主按钮）
- 忽略此行动（次要按钮）
- 已解决态：显示"已解决 · resolutionNote"
- 已忽略态：显示"已忽略 · dismissedReason"

## 六、Modal 文案

### Create Action Modal
- 标题：创建行动项
- 标签：标题（必填）/ 描述（可选）/ 分类 / 优先级
- 按钮：取消 / 创建（loading: 创建中...）
- 错误：标题不能为空 / 操作失败，请稍后重试

### Resolve Action Modal
- 标题：标记为已解决
- 显示被解决的 Action 标题
- 标签：处理说明（必填）
- Placeholder：请说明该行动如何被处理，例如已补充面评、已完成候选人风险追问、业务已确认继续推进。
- 按钮：取消 / 确认解决（loading: 提交中...）
- 错误：请填写处理说明 / 操作失败，请稍后重试

### Dismiss Action Modal
- 标题：忽略此行动
- 显示被忽略的 Action 标题
- 标签：忽略原因（必填）
- Placeholder：请说明忽略原因，例如该风险已由线下沟通处理、该行动不再适用、关联流程已关闭。
- 按钮：取消 / 确认忽略（loading: 提交中...）
- 错误：请填写忽略原因 / 操作失败，请稍后重试

## 十、已知的 UX 问题（自检清单）

1. 首屏可能像通用任务管理器，缺乏招聘领域语义
2. Demo 数据偏测试化（标题如"测试真实创建Action"）
3. sourceSummary 有些仍是技术描述而非人话
4. Linked Context 卡片缺少部门、招聘负责人等关键信息
5. Activity detail 字段直接展示 JSON，不够人话
6. Resolve/Dismiss Modal 缺少"操作可逆/不可逆"说明
7. KPI 卡片缺少业务解释
8. 逾期标识不够醒目
9. Toast 提示文字偏短，缺乏上下文
