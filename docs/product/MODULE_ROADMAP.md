## 2. 核心对象分层

```
┌──────────────────────────────────────────────────────┐
│  集成层 (Integration Layer)                           │
│  Moka / 飞书 / 外部 ATS                               │
│  Phase 9+（不做）                                     │
├──────────────────────────────────────────────────────┤
│  分析层 (Analytics Layer)                             │
│  Reports / Dashboard / AI Insight                     │
│  Phase 8.1 / 8.5 / 8.7                                │
├──────────────────────────────────────────────────────┤
│  行动层 (Action Layer)                                │
│  ActionItem / ActivityLog                             │
│  ✅ Phase 7.D 已交付                                   │
├──────────────────────────────────────────────────────┤
│  风险层 (Risk Layer)                                  │
│  RiskSignal / AIInsight / OfferRisk                   │
│  Phase 8.5 / 8.7                                      │
├──────────────────────────────────────────────────────┤
│  流程层 (Process Layer)                               │
│  Interview / InterviewFeedback / BusinessFeedback     │
│  Phase 8.4                                             │
├──────────────────────────────────────────────────────┤
│  基础层 (Foundation Layer)                            │
│  Job / Candidate / Application / Department / User    │
│  Phase 2 (已完成)                                      │
└──────────────────────────────────────────────────────┘
```

**分层原则**：
- 上层依赖下层，下层不依赖上层
- 每层可以独立测试和部署
- 基础层变更需要全量回归测试
- 分析层可以灵活调整不影响下层

## 5. Dashboard 定位说明

### Dashboard 不是 KPI 堆砌页

错误理解：
> Dashboard = 放一堆 KPI 数字 + 图表

正确理解：
> Dashboard 是招聘风险与待办聚合入口，负责把用户引导到正确的处理模块。

### Dashboard 的核心职责

| 职责 | 说明 |
|------|------|
| **聚合** | 把分散在各模块的风险、待办、动态汇聚到一个视图 |
| **引导** | KPI 卡片可点击跳转 → 告诉用户"去哪里处理" |
| **预警** | 高风险事项优先展示，不等用户自己发现 |
| **概览** | 3 秒内让用户了解招聘整体健康度 |

### Dashboard 的信息层次

```
第一层（3 秒认知）：KPI 核心数字
第二层（10 秒认知）：我的待办列表
第三层（30 秒认知）：招聘动态 + 风险提醒
第四层（点击跳转）：进入具体模块处理
```

### Dashboard 与各模块的关系

```
Dashboard 是入口，不是终点。

Dashboard 发现风险 → 点击进入 Action Center 处理
Dashboard 发现卡点 → 点击进入 Job Center 诊断
Dashboard 发现逾期 → 点击进入 Interview Center 催办
Dashboard 发现趋势 → 点击进入 Report Center 复盘
```

## 6. 不做的模块清单

| 模块 | 原因 | 替代方案 |

| 批量操作 | 逐模块实现时评估 | — |
