# 理然智能招聘 AI 看板 — 全局中文命名规范

> 版本：v1.0  
> 生效日期：2026-06-27  
> 适用范围：所有面向用户的界面文案、文档标题、系统提示  
> 原则：中文招聘语义化，拒绝英文枚举、中英混杂、通用词

---

## 一、核心原则

1. 所有面向用户的界面文案必须使用中文招聘语义
2. 禁止英文枚举值直接展示给用户（如 `pending`、`resolved`）
3. 禁止中英混杂（如 "Offer 风险中心" → 正确是 "Offer 风险"，但导航应为中文语义）
4. 禁止非招聘语义的通用词（如 "Task"、"Todo"、"Item"、"Ticket"）
5. 每个中文命名必须让 HR、面试官、管理层一眼理解其业务含义
6. 内部代码、API 字段、数据库字段不受此规范约束

## 十一、禁止使用的文案

### 全局禁止

| 禁止文案 | 原因 | 正确替代 |
|---------|------|---------|
| Coming soon!!! | 不专业，感叹号过多 | "该模块正在接入招聘数据" |
| TODO | 内部开发标记，不应暴露给用户 | 使用阶段标签如 "Phase 8.x" |
| 测试模块 | 暴露测试状态 | 使用正式模块名 + 阶段标签 |
| undefined | 程序异常 | 始终提供 fallback 值 |
| 暂无数据 | 无信息量 | "当前没有符合条件的数据" 或使用模块专属空状态 |
| Mock / Demo / Sample | 测试感 | 永远不出现 |
| Task / Todo / Item | 非招聘语义 | "行动项" / "待处理事项" |
| Open / Resolved / Dismissed | 英文枚举 | "待处理" / "已解决" / "已忽略" |

### AI 相关禁止

| 禁止文案 | 原因 | 正确替代 |
|---------|------|---------|
| AI 自动判断 | 假 AI，系统不做自动判断 | "系统规则提醒" |
| AI 决定录用 | 虚假 AI 决策 | 永不使用 |
| AI 自动淘汰 | 虚假 AI 决策 | 永不使用 |
| AI 自动推进 | 虚假 AI 决策 | 永不使用 |
| GPT 生成 | 未接入 LLM 时禁用 | "AI 辅助建议" |
| OpenAI 已生成 | 未接入 LLM 时禁用 | "AI 辅助建议" |
| 智能定薪 | 过度承诺 | "薪资建议（仅供参考）" |
| 一键生成+确认+推送 | 暗示自动化 | 分别操作：生成 → 确认 → 推送 |

### 风格禁止

| 禁止风格 | 原因 | 正确风格 |
|---------|------|---------|
| 作战/战报/军情 | 军事化隐喻 | 招聘复盘/数据概览 |
| 火力/攻坚/突击 | 军事化隐喻 | 重点推进/优先处理 |
| 指挥中心 | 军事化隐喻 | 工作台/招聘总览 |
| 面霸 | 不尊重的标签 | 不使用 |
| 面试官排名 | 不尊重的表述 | 面试官质量评估 |
| 大屏/驾驶舱 | 传统 BI 风格 | 招聘总览 |

## 十二、文案模板

### 页面标题与描述

```
格式：{模块中文名}
描述：{一句话说明这个页面解决什么招聘问题}
```

示例：
- **招聘总览（AI 招聘洞察看板）** — 基于招聘过程数据，辅助识别招聘风险、流程卡点和优先处理事项
- **风险行动中心** — 所有招聘风险的统一处理入口，从识别到闭环
- **岗位** — 岗位全生命周期管理，追踪招聘进度与卡点诊断
- **候选人** — 候选人全链路追踪，推进或淘汰的决策依据
- **面试管理** — 面试安排与反馈管理，确保每次面试有据可查

### 空状态

```
标题：该模块正在接入招聘数据
描述：当前可先通过招聘总览和风险行动中心查看核心招聘风险、待处理事项和行动闭环。
```

### 权限拒绝

```
标题：权限不足
描述：您没有权限访问此页面。如需访问请联系管理员。
```

### 错误状态

```
标题：加载失败
描述：请稍后重试或联系管理员
```

### 加载状态

```
不展示文案，使用骨架屏（Skeleton）替代 Spinner。
```

## 十三、命名决策流程

当需要新增一个面向用户的文案时，按以下流程决策：

1. **这是招聘业务概念吗？** → 使用招聘语义中文
2. **有现成的中文术语吗？** → 优先使用行业标准中文术语
3. **本表是否已有定义？** → 直接使用本表定义
4. **是否需要新增？** → 提交 PR 更新本表，经 Review 后生效
