## 1. 状态视觉区分审查

### 设计要求

6 种集成状态必须通过颜色、图标、文案三个维度进行视觉区分，确保用户无需阅读详细文本即可快速识别状态。

### 实现验证

| 状态 | 颜色类 | 视觉色值 | 图标 | 中文文案 | 视觉效果评估 |
|------|--------|----------|------|----------|-------------|
| 已配置 · 可用 | `text-green-600` | 🟢 绿色 | ✅ | 已配置 · 可用 | ✅ 清晰，正面含义 |
| 未配置 · 等待连接 | `text-gray-500` | ⚪ 灰色 | ⚙️ | 未配置 · 等待连接 | ✅ 中性，提示操作 |
| 需授权 · 配置后可读取 | `text-orange-500` | 🟠 橙色 | 🔐 | 需授权 · 配置后可读取 | ✅ 警示，需关注 |
| 只读预留 · 不写回 | `text-blue-600` | 🔵 蓝色 | 👁️ | 只读预留 · 不写回 | ✅ 信息性，特殊状态 |
| 连接异常 | `text-red-600` | 🔴 红色 | ❌ | 连接异常 | ✅ 错误，需立即处理 |
| 连接超时 | `text-yellow-500` | 🟡 黄色 | ⏱️ | 连接超时 | ✅ 警告，需重试 |

### Badge 组件变体映射

| 状态 | Badge variant | 视觉效果 |
|------|--------------|----------|
| 已配置 · 可用 | `success` | 绿色背景 + 白色文字 |
| 未配置 · 等待连接 | `secondary` | 灰色背景 + 深色文字 |
| 需授权 · 配置后可读取 | `warning` | 橙色背景 + 深色文字 |
| 只读预留 · 不写回 | `info` | 蓝色背景 + 白色文字 |
| 连接异常 | `destructive` | 红色背景 + 白色文字 |
| 连接超时 | `warning` | 黄色背景 + 深色文字 |

### 颜色无障碍检查

- 红/绿色盲友好：每种状态同时使用图标 + 颜色 + 文字，三通道信息冗余
- 对比度：所有 Badge 变体均使用 Tailwind 标准配色，符合 WCAG AA 标准
- 暗色模式：Tailwind `dark:` 变体已适配

**结论**: ✅ 通过 — 6 种状态在颜色、图标、文案三个维度均有清晰区分

## 5. 密钥掩码审查

### 掩码规则

| 输入示例 | 输出 | 规则 |
|----------|------|------|
| `sk-abc123def456ghi7` | `••••••••hi7` | 前 8+ 位掩码 + 后 4 位明文 |
| `sk-short` | `••••••••` | 长度不足 8 位全掩码 |
| `feishu_app_secret_long` | `••••••••ong` | 通用掩码规则 |

### UI 展示

```
┌─────────────────────────────────────────────┐
│  API Key                                     │
│  ┌─────────────────────────────────────┐     │
│  │ ••••••••ghi7                        │     │
│  └─────────────────────────────────────┘     │
│  ℹ️ 密钥已加密存储，仅用于 API 调用            │
│  [显示] [复制] [重置]                         │
└─────────────────────────────────────────────┘
```

### SecretInput 组件行为

| 操作 | 行为 |
|------|------|
| 默认状态 | 显示掩码值 `••••••••xxxx` |
| 点击"显示" | 临时显示完整密钥（5 秒后自动掩码） |
| 点击"复制" | 复制完整密钥到剪贴板，toast 提示"已复制" |
| 悬停 | tooltip 显示后 4 位 |
| 离开页面 | 自动恢复掩码状态 |

**结论**: ✅ 通过 — 密钥默认掩码显示，复制/查看功能有超时保护

## 9. 状态徽章颜色映射验证

### Badge 组件颜色配置

```typescript
// src/features/integration/status-badge.tsx
const BADGE_VARIANT_MAP: Record<IntegrationStatus, BadgeVariant> = {
  configured_available: "success",    // 绿色
  not_configured: "secondary",       // 灰色
  permission_required: "warning",    // 橙色
  readonly_reserved: "info",         // 蓝色
  connection_error: "destructive",   // 红色
  connection_timeout: "warning",     // 黄色
};
```

### 颜色语义验证

| 状态 | Badge 颜色 | 语义含义 | 语义合理性 |
|------|-----------|----------|-----------|
| 已配置 · 可用 | 绿色 | 成功/正常 | ✅ 符合通用认知 |
| 未配置 · 等待连接 | 灰色 | 中性/待处理 | ✅ 不引起焦虑 |
| 需授权 · 配置后可读取 | 橙色 | 警告/需关注 | ✅ 提醒但不恐慌 |
| 只读预留 · 不写回 | 蓝色 | 信息/提示 | ✅ 信息性状态 |
| 连接异常 | 红色 | 错误/需处理 | ✅ 引起重视 |
| 连接超时 | 黄色 | 警告/可恢复 | ✅ 临时性问题 |

### 颜色一致性

- 所有绿色状态表示"正常可用"
- 所有红/橙色状态表示"需要用户干预"
- 蓝色保留给特殊状态（只读）
- 灰色用于中性/初始状态

**结论**: ✅ 通过 — 状态徽章颜色映射语义正确，与用户认知一致
