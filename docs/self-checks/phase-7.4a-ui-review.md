# Phase 7.4A — UI Review

## 自检清单

| 检查项 | 结果 | 说明 |
|--------|------|------|
| 页面是否像招聘 SaaS | 是 | 浅灰背景 + 白色卡片 + 克制 Badge + 招聘语境关联对象 |
| 页面是否像普通任务管理器 | 否 | 关联对象展示候选人/岗位/面试，非通用 task |
| 页面是否像传统后台 | 否 | 无粗边框表格、无大面积灰色、无传统 CRUD 表单 |
| KPI 是否有业务含义 | 是 | 待处理/逾期/高优先级/今日到期/平均关闭时长/按时关闭率 |
| Badge 是否克制 | 是 | 圆角 pill + 浅色背景 + 小字体 |
| 风险色是否小面积使用 | 是 | 仅 overdue 文字用红色，非大面积色块 |
| 列表信息是否完整 | 是 | 标题/摘要/分类/优先级/状态/负责人/关联对象/来源/截止/更新 |
| 筛选是否清晰 | 是 | SegmentedControl 分段控制器，状态/优先级/分类/逾期 |
| 空状态是否招聘语境化 | 是 | "招聘流程中的风险跟进、反馈催办和协同事项会出现在这里" |
| 错误状态是否隐藏技术细节 | 是 | 仅显示错误信息 + 重试按钮，无 stack trace |
| 权限状态是否不泄露无权限对象 | 是 | 通用权限说明，不显示对象数量 |

## 参考来源

| 产品 | 学习的特征 | 落地 |
|------|----------|------|
| Panda Design System | Dashboard KPI 卡片 + 白色卡片 | KPI Cards 布局 |
| Linear | 列表密度 + 低噪音 | ActionList 表格密度 |
| Greenhouse | 招聘流程结构化 | 关联对象展示 (candidate > job > interview) |
| Tremor | KPI 数字格式 | 数值 + hint 标签 |
